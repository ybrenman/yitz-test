Overview of Installation

Installation of Pivotal Analytics requires working with your internal networking environment, a VMware vSphere ESXi environemnt, a Pivotal HD installation, and the Pivotal Analytics software. The Main Installation Steps table below describes the the installation process and provides links to the procedures you should follow. 

When you install Pivotal Analytics, you first deploy a single virtual machine (VM) in a VMware vSphere environment. This single VM contains pre-installed Pivotal Cloud Foundry software that enables the installation of Pivotal Analytics. You access this software using a Web browser with a network connection to this single VM. This installation software application allows you to describe server, Hadoop, networking, authentication, and other configuration parameters for your installation. The software then provisions and deploys multiple VMs in the vSphere environment based on the information you provide.

Before you begin installing Pivotal Analytics, make sure you have the required hardware and software available. See System Requirements and Sizing Guidelines.

4. Prepare a vSphere ESX cluster for use with Pivotal Anatlytics

You deploy Pivotal Analytics on one or more VMware ESXi server(s). The size and number of ESX servers required depend on the amount of data and type of analysis you need to perform.

You use a vSphere client to configure your ESXi environment for use with Pivotal Analytics. See your VMware vSphere documentation for more information.

For help with determining the correct number and size of servers, see Sizing Guidelines.

Configuring the vSphere Environment
5. Deploy the Pivotal CF Installer template as a virtual machine

You use a OVA template file to create a virtual machine that runs the Pivotal CF Installer. You use this installer to install Pivotal Analytics, and optionally, Cloud Foundry services.

After you configure and start this virtual machine, the Pivotal CF Installer application starts. You access this application using a Web browser that connects to services running on this virtual machine.

Deploying the Pivotal CF Virtual Machine
6. Run the Pivotal CF Installer

Using the Pivotal CF Installer Web application, you enter configuration values for the Pivotal CF Installer.

Configuring the Pivotal CF Installer to Install Pivotal Analytics
7. Configure Pivotal Analytics

Using the Pivotal CF Installer Web application, you enter configuration values for your Pivotal Analytics installation. The Pivotal CF Installer uses this configuration information and connects to your vSphere cluster and provisions the virtual machines needed for your deployment, installs the Pivotal Analytics software, and starts the application. Depending on the size of your deployment this step may take up to several hours to complete and does not require user interaction.

Required Information for Installation

You will need the following information available before you begin the installation:

  • IP Address of the vCenter where you will deploy Pivotal Analytics. 
  • SMTP (outgoing mail server) connection information (IP address, port number, user name, and password)
  • Proxy server connection information (needed only if proxy servers are required in your network) 
  • Pivotal HD connection information IP addresses and port numbers for the following Pivotal HD components:
    • HDFS namenode
    • Hbase master
    • Hive server
    • Jobtracker
    • Yarn resource manager
    • Yarn scheduler
    • Map Reduce History Server
    • Host mappings?? for which things?
  • Networking information. See Networking Requirements.
  • HDFS endpoint connection information 
  • YARN resource manager connection information 
  • Hbase master connection infomration
  • Name and email address of at least one Administrative user
  • Email alias for user support
  • Email address to receive new user notifications.

To being installation of Pivotal Analytics, begin with Configuring the vSphere Environment.
